## md2docx.sh

### 问题 1: `read` 在管道/重定向时失效 [中]

```bash
echo "node_modules not found. Install dependencies now? [Y/n]"
read -r answer
```

当脚本通过管道调用时（如 CI 环境），`read` 从 stdin 读取会失败。应使用 `/dev/tty`：

```bash
read -r answer < /dev/tty
```

### 问题 2: `npm install` 失败未处理 [高]

```bash
(cd "$PROJECT_DIR" && npm install)
```

`npm install` 失败时（网络问题、权限不足），脚本没有检查退出码，会继续执行后续逻辑。应加 `|| exit 1`：

```bash
(cd "$PROJECT_DIR" && npm install) || exit 1
```

### 问题 3: `node` 存在性检查缺失 [低]

脚本假设 `node` 命令已存在于 PATH 中，但未验证。在部分环境中可能导致难以理解的错误信息。

建议添加：

```bash
if ! command -v node >/dev/null 2>&1; then
  echo "ERROR: Node.js is not installed or not in PATH"
  exit 1
fi
```

### 不是问题

- **`set -euo pipefail`** — 不适用。脚本需要处理多个文件，单个文件失败应 `continue` 而非退出。
- **`failed` 数组重复添加** — 不存在。文件不存在（第 50 行）和 preprocess 失败（第 62 行）是互斥场景。

## preprocess.js

### 问题 1: `js-yaml` 是隐式依赖 [中]

```javascript
const yaml = require('js-yaml');
```

`package.json` 中没有 `js-yaml`，虽然 `gray-matter` 间接引入了它，但隐式依赖存在风险：
- 上游包升级后可能不再依赖 `js-yaml`
- 不同包管理器的依赖提升行为不一致

应在 `package.json` 中显式声明：

```json
"dependencies": {
  "js-yaml": "^4.1.0"
}
```

### 问题 2: `checkDeepHeadings` 未使用 [低]

第 91-101 行定义了 `checkDeepHeadings`，但在 `main()` 中未调用。这是死代码，建议删除或接入主流程。

### 问题 3: `stripBulletManualNumbers` 正则覆盖不全 [低]

```javascript
const m = line.match(/^(\s{0,6})[-*]\s+([a-z]\)|\(\d+\))\s+(.+)$/);
```

仅匹配 `a)` 和 `(1)` 形式，不匹配 `1)`、`A)` 等。如果用户用 `1)` 作为手动编号，无法被剥离。

建议扩展为：

```javascript
const m = line.match(/^(\s{0,6})[-*]\s+([a-zA-Z]\)|\(\d+\)|\d+\))\s+(.+)$/);
```

### 不是问题

- **YAML 去重逻辑** — 对于该项目的使用场景（简单 front matter）足够健壮。
- **`HEADING_NUM_SINGLE` 正则** — 注释与实际逻辑一致，`5G 网络` 等场景已正确处理。
