## 问题 1：缺少对图注（Figure Caption）的考虑 [中]

**位置：** 方案正文

方案只提到了"表格题注（Caption）"，但忽略了"图片题注（Figure Caption）"。

当前代码中：
- 表格题注使用 `**表 X-X 名称**` 格式，被识别为 Caption 样式
- 图片题注（如 Mermaid/PlantUML 渲染后的图注）也使用 `**图 X-X 名称**` 格式，同样被识别为 Caption 样式

**影响：** 如果图片题注和图片分页，方案没有覆盖。

**建议：** 在方案中明确说明：
- Caption 样式同时用于表格题注和图片题注
- 修复后，表格题注与表格同页，图片题注与图片同页

## 问题 2：缺少对 `keepNext` 副作用的考虑 [低]

**位置：** 方案正文

`keepNext` 有一个副作用：如果 Caption 段落很长（多行），且后面紧跟着表格，而表格又很大，可能导致整页内容被挤到下一页，留下大量空白。

**场景：**
```
[页尾]
...正文...
表 3-1 系统接口列表（很长很长的题注文字...）
[大表格]

如果 Caption + 表格 > 剩余页面空间，
则 Caption 会整体移到下一页，
当前页留下大量空白。
```

**建议：** 这是 Word 的正常行为，无需处理。但应在方案中说明这是预期行为。

## 问题 3：缺少对 "表格跨页时表头重复" 的确认 [低]

**位置：** 方案正文

当前代码中：
```javascript
const headerRow = new TableRow({
  tableHeader: true,
  // ...
});
```

`tableHeader: true` 已经设置了表头行重复。但 `patchDocxPagination` 中又给表头行首段落加了 `keepNext`。

**问题：** `tableHeader: true` 和 `keepNext` 是否有冲突？

**分析：**
- `tableHeader: true` — Word 自动在跨页时重复表头
- `keepNext` — 表头行首段落与下一段（数据行）保持同页
- 两者不冲突：`tableHeader` 处理跨页重复，`keepNext` 处理表头与第一行数据的关系

**建议：** 在方案中说明两者不冲突。

> 2. 在 Word 中查看表格题注和表格是否同页

**问题：** 没有说明如何"在 Word 中查看"。

**建议：** 补充具体步骤：
> 1. 生成包含表格的 DOCX
> 2. 在 Word 中打开，切换到"视图" → "草稿"或"页面视图"

> 6. 也可以右键点击题注段落 → "段落" → "换行和分页"，查看"与下段同页"是否勾选
