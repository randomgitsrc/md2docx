## Commands

```bash
# Install dependencies (run once)
npm install

# Full pipeline: preprocess then convert a single file
node scripts/preprocess.js md/input.md && node scripts/md2docx.js md/output/clean/input.clean.md

# Or use the shell wrapper (recommended for batch processing)
./scripts/md2docx.sh md/input.md

# Preprocess only (fixes YAML, strips heading numbers, renders mermaid, marks captions)
node scripts/preprocess.js md/input.md

# Convert preprocessed markdown to DOCX
node scripts/md2docx.js md/output/clean/input.clean.md

# Generate a blank template DOCX (reference only, hardcoded sample content)
node scripts/build_template.js
```

Output goes to `md/output/docx/` by default. Mermaid PNGs are cached in `md/output/.mermaid/`.

## External Dependencies

- **mmdc** (mermaid-cli) вЂ” renders `mermaid` code blocks to PNG. Installed as npm dep, invoked via `npx mmdc`.
- **python3 + python-docx** вЂ” post-processes the generated DOCX to inject pagination XML attributes (`cantSplit`, `keepNext`, `keepLines`) that the `docx` npm package cannot set natively.
- **Chrome/Chromium** вЂ” required by mmdc for headless rendering. `puppeteer-config.js` auto-detects the browser path.

## Architecture

The pipeline has two stages, run sequentially:

### Stage 1: `scripts/preprocess.js` вЂ” Markdown cleanup

Reads raw `.md`, applies transformations, writes `output/clean/*.clean.md`:

1. **fixYamlFrontMatter** вЂ” deduplicates YAML keys, migrates `author`в†’`company`, `doc_title`в†’`title`
2. **stripHeadingNumbers** вЂ” removes manual numbering from headings (`## 1 иЊѓе›ґ` в†’ `## иЊѓе›ґ`)
3. **renderMermaidBlocks** вЂ” calls `mmdc` to render ` ```mermaid ` blocks to PNG, replaces with `![](path)` references
4. **stripBulletManualNumbers** вЂ” removes manual list numbering (`- a) text` в†’ `- text`)
5. **markCaptions** вЂ” wraps `иЎЁ X-X еђЌз§°` / `е›ѕ X-X еђЌз§°` lines in `**bold**` for the converter to detect

### Stage 2: `scripts/md2docx.js` вЂ” Markdown в†’ DOCX

`Md2DocxConverter` class parses markdown-it tokens into `docx` library objects:

- **Headings**: `#` = cover title (stripped), `##`вЂ“`######` = Heading 1вЂ“5 with auto multi-level numbering
- **Lists**: 3-level nested list pool (200 instances each) so each top-level list restarts numbering from 1
- **Tables**: auto-sized columns by content width, header row repeats across pages, bold headers
- **Images/Mermaid**: PNG embedded with `fitImageToPage` scaling (constrained to content area)
- **Captions**: detected by `**е›ѕ/иЎЁ X-X**` pattern, styled as centered й»‘дЅ“ е°Џе››
- **Code blocks**: monospace Consolas, дє”еЏ· font

Document is assembled as 3 Word sections: Cover в†’ TOC (Roman numerals) в†’ Body (Arabic page numbers from 1).

`patchDocxPagination()` runs after DOCX generation, calling python-docx to inject XML pagination properties.

## Key Conventions

- `#` (H1) in markdown is exclusively the document title for the cover page. Section headings start at `##`.
- Heading numbering is auto-generated by Word вЂ” never write manual numbers in markdown headings.
- Captions must follow the `**е›ѕ X-X еђЌз§°**` or `**иЎЁ X-X еђЌз§°**` pattern (preprocess.js adds the bold markers).
- List items must use `-` syntax; manual numbering prefixes are stripped by the preprocessor.
