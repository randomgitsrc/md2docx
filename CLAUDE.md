## Project Overview

md2docx is a Markdown-to-DOCX converter that produces Chinese technical documents conforming to the 《技术文档格式-20260525》 specification. It converts `.md` files with YAML front matter into Word documents with cover page, table of contents, and styled body sections.

## Commands

```bash
# Full conversion pipeline: preprocess then convert
node scripts/preprocess.js md/input.md && node scripts/md2docx.js md/output/clean/input.clean.md

# Preprocess only (fixes YAML, strips heading numbers, renders mermaid, marks captions)
node scripts/preprocess.js md/input.md

# Convert preprocessed markdown to DOCX
node scripts/md2docx.js md/output/clean/input.clean.md

# Generate a blank template DOCX (reference only, uses hardcoded content)
node scripts/build_template.js
```

Output goes to `md/output/docx/` by default. Mermaid PNGs are cached in `md/output/.mermaid/`.

## External Dependencies

- **mmdc** (mermaid-cli) — renders `mermaid` code blocks to PNG. Installed as npm dep, invoked via `npx mmdc`.
- **python3 + python-docx** — post-processes the generated DOCX to inject pagination XML attributes (`cantSplit`, `keepNext`, `keepLines`) that the `docx` npm package cannot set natively.
- **Chrome/Chromium** — required by mmdc for headless rendering. `puppeteer-config.js` auto-detects the browser path.

## Architecture

The pipeline has two stages, run sequentially:

### Stage 1: `scripts/preprocess.js` — Markdown cleanup

Reads raw `.md`, applies transformations, writes `output/clean/*.clean.md`:

1. **fixYamlFrontMatter** — deduplicates YAML keys, migrates `author`→`company`, `doc_title`→`title`
2. **stripHeadingNumbers** — removes manual numbering from headings (`## 1 范围` → `## 范围`)
3. **renderMermaidBlocks** — calls `mmdc` to render ` ```mermaid ` blocks to PNG, replaces with `![](path)` references
4. **stripBulletManualNumbers** — removes manual list numbering (`- a) text` → `- text`)
5. **markCaptions** — wraps `表 X-X 名称` / `图 X-X 名称` lines in `**bold**` for the converter to detect

### Stage 2: `scripts/md2docx.js` — Markdown → DOCX

`Md2DocxConverter` class parses markdown-it tokens into `docx` library objects:

- **Headings**: `#` = cover title (stripped), `##`–`######` = Heading 1–5 with auto multi-level numbering
- **Lists**: 3-level nested list pool (200 instances each) so each top-level list restarts numbering from 1
- **Tables**: auto-sized columns by content width, header row repeats across pages, bold headers
- **Images/Mermaid**: PNG embedded with `fitImageToPage` scaling (constrained to content area)
- **Captions**: detected by `**图/表 X-X**` pattern, styled as centered 黑体 小四
- **Code blocks**: monospace Consolas, 五号 font

Document is assembled as 3 Word sections: Cover → TOC (Roman numerals) → Body (Arabic page numbers from 1).

`patchDocxPagination()` runs after DOCX generation, calling python-docx to inject XML pagination properties.

### Supporting files

- `scripts/puppeteer-config.js` — shared Chrome path detection for mmdc
- `scripts/build_template.js` — standalone template generator with hardcoded sample content (reference implementation, not part of the pipeline)
- `01-base/技术文档格式-20260525.md` — the formatting spec all styles conform to

## Key Conventions

- `#` (H1) in markdown is exclusively the document title for the cover page. Section headings start at `##`.
- Heading numbering is auto-generated by Word — never write manual numbers in markdown headings.
- Captions must follow the `**图 X-X 名称**` or `**表 X-X 名称**` pattern (preprocess.js adds the bold markers).
- List items must use `-` syntax; manual numbering prefixes are stripped by the preprocessor.
- Fonts: 仿宋 (body), 黑体 (Heading1/Caption), 楷体 (Heading2), 方正小标宋简体 (cover title), Times New Roman (Western text), Consolas (code).
